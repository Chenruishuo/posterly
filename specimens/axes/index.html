冲突规则(组合时校验)

硬规则 —— 校验,不辩论

  1. 容量 gate:内容量 × 版式 × 密度必须满足最低字号/padding;塞不下就换轴或裁内容,绝不缩正文。
  2. 结构前提:fieldset / 骑线胶囊需要真实边框(frame-line ≠ none);侧标签需要预留边距;竖排侧轨需要横带行结构与轨道宽度。
  3. 页头与版式一致:巨题列刊头 → 版式必须有标题轨道;题入色板 → 版式必须有舞台区。
  4. neon → 低亮度底 + 发光描边;白底禁用。
  5. 发光效果只在局部背景为暗时启用(白海报里的一块暗板,发光只许出现在板内)。
  6. 透明卡面:文字对比按实际底面最差点校验(分区/渐变底也算);暗底上的论文插图要加浅色衬板或反相。
  7. 语义贯穿:承载语义的色全页不变;categorical 配色过色差与色盲检查;占用的语义色不得挪作装饰。
  8. 外伸构造(连体 chip / 悬浮胶囊)防 overflow 截断、防与上行元素相撞;嵌套内层框要在全局关边框时幸存。
  9. 渐变/色板上的文字:对比按文字覆盖区的最差点计算,不用平均色。

软规则 —— 合法,但要权衡

  • 暗底可以有阴影/发光(只避免低对比灰投影,66072 为证)。
  • 无边框布局需要某种替代分组信号(间距、规则线、标题条、对齐皆可,不一定是更大间距)。
  • 少见但合法:sparse + 4 栏、奶油底 + 怪诞体、tonal 配色 + 巨数字墙。

反趋同

  • 同一批连续海报必须在「底色 / 框线 / 节标题接合 / 页头」中至少两轴不同,且不得沿用上一张的概念陈述;出厂默认(白底·软卡·素题·居中页头)也算一种组合,不许每张都塌缩回去。

七个命名配方(宣言色板 / 纸感学术 / 暗夜编辑 / 奶油刊头 / 工程报告 / 机构单色 / 粉彩圆角)定位是「各轴默认值包」——起点,不是套餐;逐轴的完整配置尚未收录,目前每个配方的名字先当一句现成的概念陈述用(SKILL.md Step 2.5):它说出海报住在哪个世界,八轴自行组合。等配置收录后,换掉其中任何一轴也都正常,换完确认新选择仍服务概念、再重查上面的规则即可。
