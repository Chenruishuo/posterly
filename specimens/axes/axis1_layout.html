本轴的一次选择是一个结构化对象,不是单选枚举: base topology(基础拓扑,下列 10 选 1) + span modifier(跨列修饰:none / top-hero / 全宽中段横带) + focal content(焦点区放什么:figure / statement / metric wall / title,选型时必须写明)。

竖版翻译 Portrait translations画布 24:36 · 横版拓扑的竖版形态 · P1–P7

朝向是会场输入。本节按 ~24 in 宽的竖版档位标定(24×36 in 板及几乎同尺寸的 61×91 cm ICLR workshop 规格):可用宽度只有约 2 栏,便宜方向是全宽行, 读者默认自上而下扫读;板子总量也小 — 面积仅为 60×36 横版的 ~40%。 A0 竖版(33.1×46.8 in)是另一个档位,这些英寸数不通用:3 栏 ~10.4 in 已舒适、 4 栏 ~7.7 in 降到 ⚠ 档、并列 hero 支撑栏 ~12.7 in 可行 — 见规范表的 A0 条目。 翻译原则:保留拓扑的想法,把焦点几何搬到全宽行上 — 不要临场发挥, 也不要让每张竖版都塌回 03 双栏纵流(防趋同的指纹轴含版式骨架)。 判定汇总:03 原生适用;07 / 09 / 05 / 08 / 06 / 10 的竖版形态见 P1–P7; 04 英雄面板并列形态 ⚠ 合法但戏剧性弱(支撑栏 ~9 in 只装短卡,hero 轨 ~13.6 in 比普通双栏宽不了多少),通常改用 P2 / P3 的横带形态; 02 三栏 ⚠ 栏宽仅 ≈7.3 in,只装得下短句 / 列表;01 四栏 ✗ 低于最小可读栏宽。 锚点均为横版原型(尚无竖版语料);规范表见 templates/DESIGN-AXES.md §Axis 1 · portrait translations。 本节契约与横版节不同:线框画的是区域几何示意;代码块给出的才是 门禁角色合规的可搬运构造(外层 poster grid + 内层 column / banner 结构), 两者的行分割不必逐字相同。 measure 角色:column 只给底边收在同一线上的贯底卡栈 (带堆叠 = 一条全宽 column,各带标 card;spread 按全部 column / hero 底边计算);全宽 hero / 舞台带标 banner(不计入 spread — wave-3 天象仪带先例),不要标 hero(hero 语义是与栏底对齐的贯底面板); 中段带上方的支撑区同样不能标 column,整块包进 banner; card 只能挂在 column / hero 下(preflight),banner 内部构件不带角色。 注意两点:banner 必须直接挂在 poster 下(不能在 body 角色区内); banner 内容不受裁剪 / 坏图 / letterbox 门禁覆盖(那些只扫 card / column / hero)— 带内视觉须以 print 仿真渲染(render_preview 的 PDF/PNG,非屏幕预览)人工过目。

/* 外层 poster grid */
grid-template-rows:    auto minmax(0, 1fr) auto;
grid-template-areas:   "ht" "bd" "ft";
/* 内层:bd = body 角色 2 栏 grid,两条贯底 column;
   两栏共用 7 条等高半行轨道(轨道高 = (body 高 − 6×行 gap,即 6×7u) / 7,
   随画布缩放;右栏错开一个网格行 = 一条轨道 + 一个行 gap),
   普通卡跨 2 个半行、左右错 1 个 — 错位与末端对齐都由轨道保证: */
.spine-body  { position: relative; display: grid;
               grid-template-columns: 1fr 1fr;
               gap: calc(12 * var(--u)); min-height: 0; }
.spine-col   { display: grid; gap: calc(7 * var(--u)); min-height: 0;
               grid-template-rows: repeat(7, minmax(0, 1fr)); }
.spine-col--l > :nth-child(1) { grid-row: 1 / 3; }  /* 左:1/3, 3/5, 5/8 */
.spine-col--l > :nth-child(2) { grid-row: 3 / 5; }
.spine-col--l > :nth-child(3) { grid-row: 5 / 8; }
.spine-col--r > :nth-child(1) { grid-row: 2 / 4; }  /* 右:2/4, 4/6, 6/8 */
.spine-col--r > :nth-child(2) { grid-row: 4 / 6; }
.spine-col--r > :nth-child(3) { grid-row: 6 / 8; }
/* :nth-child 按「每栏恰有 3 个直接子元素且全是 card」假设计数 —
   栏内一旦加入其他直接子级(标签 / 装饰),槽位整体错移;
   那种情况改用显式槽位类(.sp-1/.sp-2/.sp-3)标 grid-row。
   卡数不同时改行数与 grid-row 放置,保持两栏末行同线收尾;
   卡内容要能装进其轨道高(容量按卡预算) */
/* 脊线画在 .spine-body 上 — 不要用 .poster::before,
   竖版模板已用它画顶部色条,两者会互相覆盖: */
.spine-body::before {
  content: ""; position: absolute;
  left: 50%; top: 0; bottom: 0;
  width: calc(1.2 * var(--u)); transform: translateX(-50%);
  background: var(--border-strong);
}

适用场景方法真有"一条主线 + 多个环节"时:竖版的中心—辐射 — 主线变成沿中缝的脊线,环节卡片左右交替错落。
