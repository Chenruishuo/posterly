轴 2 — 底色 Canvas base + treatment + coverage

一个画布配置由三个子选择自由组合而成,不是互斥枚举: base(铺什么颜色)× treatment(颜色怎么处理:平涂 / 渐变 / 纹理装饰)× coverage(铺到哪里:全幅 / 分区双色 / 只露缝作框)。

归属边界:整页级装饰(星图点缀、整页饰框)属于本轴的 treatment=textured-decorated;单个模块的装饰边框属于轴 6 (frame-line: ornamental)。分区色与缝色仍取自 base —— coverage 只决定几何,不引入新的颜色角色。

锚点 ID 指 80 张 ICML 2026 精选海报,可按 icml.cc/media/PosterPDFs/ICML%202026/<id>.png 重看原构造。 每个选项演示为一张 60:36 迷你海报;占位卡为全页统一的中性白卡,便于横向比较底色本身的效果。

base · 底色本体(5) treatment · 处理方式(3) coverage · 覆盖方式(3)

base · 底色本体5 options

本节控制变量:treatment=flat、coverage=full,只换底色。

treatment · 处理方式3 options

本节控制变量:base 固定为奶油(cream)、coverage=full,只换处理方式。 注:60839 原作的星图铺在品牌色底上,此处按单一变量原则移到奶油底,演示的是"低调整页纹理"这一手法本身。

coverage · 覆盖方式3 options

本节控制变量:treatment=flat。full 与 split 以白为基; gutter-as-frame 的缝色必须可见(白底会让"框"消失),故直接使用锚点原色 —— 缝色本身就是 base 的取值,coverage 只改变几何。

控制变量说明:base 节固定 treatment=flat / coverage=full;treatment 节固定 base=cream / coverage=full; coverage 节固定 treatment=flat。占位卡为全页统一的中性白卡(标题条 + 文本条 + 图占位),只提供"有内容"的语境。

搬运指引:画布构造集中在 .ax2-mini--* 修饰类与 :root 底色令牌 (CSS 内有 [搬运] 标记),不依赖占位卡结构,可直接移入组件库。
